# Salon CMS

Сучасний, кастомний CMS-движок для салону краси, розроблений на PHP (без використання важких фреймворків) із фокусом на швидкість, простоту та гнучкість.

## 🚀 Як це працює?

Проєкт використовує модульну архітектуру. Весь статичний контент, тексти на різних мовах, ціни та налаштування зберігаються у базі даних, а не «зашиті» (hardcoded) у файли верстки.

- **Frontend (Клієнтська частина):** Розташований у папці `pages/`. Головна сторінка знаходиться в `pages/home.php`. Дані на сторінку виводяться за допомогою спеціальних функцій-хелперів (наприклад, `tr('hero_title')`), які підтягують переклади з бази залежно від обраної мови (`$_SESSION['lang']`).
- **Backend (Адмін-панель):** Розташована у папці `admin/`. Має захищену паролем систему авторизації (`index.php`), управління текстами та SEO-налаштуваннями (`pages.php`), а також гнучке управління прайс-листом на базі 3-рівневої ієрархії: **Категорія ➝ Підкатегорія ➝ Послуга** (`services.php`).
- **База Даних:** Всі запити до БД проходять через клас-обгортку `DB.php`, який використовує безпечний PDO інтерфейс із підготовленими запитами.

---

## 🛠 Технології

- **Backend:** PHP 8.3
- **База Даних:** MariaDB 11.2 (MySQL)
- **Frontend CSS:** Tailwind CSS (вказано класи), кастомний CSS (`assets/css/theme.css`).
- **Frontend JS:** Vanilla JS (без jQuery), Fancybox (для галереї/слайдерів).
- **Роутинг:** Кастомний, на базі файлу `index.php` у корені.
- **Локальне середовище:** Open Server Panel (OSPanel)

---

## ⚙️ Що потрібно для запуску

1. Веб-сервер із підтримкою **PHP 8.3+** (рекомендується Open Server Panel).
2. Сервер баз даних **MariaDB 11.2** або сумісний (MySQL 8+).
3. Домен/хост (наприклад, `nox`), налаштований на корінь проєкту.

---

## 🚀 Як підняти проєкт (Локально на OSPanel)

1. Помістіть усі файли проєкту у директорію `domains/nox/` (або `home/nox/` залежно від версії OSPanel).
2. Запустіть Open Server Panel.
3. Перевірте, чи увімкнені відповідні модулі в OSPanel:
   - PHP: **PHP-8.3**
   - База даних: **MariaDB-11.2**
   *(Ці налаштування також автоматично підтягуються з файлу `.osp/project.ini`)*.
4. Відкрийте браузер і перейдіть за адресою: `http://nox`

### Налаштування Бази Даних

Підключення до бази даних налаштовується у файлі **`config.php`**, який знаходиться у корені проєкту.

Актуальні налаштування для локального запуску:
```php
'db' => [
    'host' => 'MariaDB-11.2',
    'dbname' => 'salon_cms',
    'user' => 'root',
    'password' => '',
    'charset' => 'utf8mb4'
]
```
*(Якщо ви переносите сайт на інший хостинг, змініть ці дані на надані хостинг-провайдером).*

## 📂 Структура проєкту

- `/admin/` — Адміністративна панель.
- `/assets/` — Статичні файли: зображення, стилі (CSS), скрипти (JS).
- `/components/` — Повторювані частини HTML (хедер, футер).
- `/pages/` — Основні сторінки сайту (наприклад, `home.php`).
- `/config.php` — Основний файл конфігурації проєкту (доступи до БД та пошти).
- `/DB.php` — Клас для роботи з базою даних.
- `/index.php` — Точка входу сайту, роутер.
- `AGENT_RULES.md` — Внутрішні правила розробки (для ШІ).
